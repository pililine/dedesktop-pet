# New Codex Onboarding

Last updated: 2026-07-08

This is the handoff guide for a fresh Codex session after the old image-generation conversation is deleted.

## Open The Project Root

Open the Desktop Pet Factory repository root: the directory that contains all of the following:

```text
Makefile
README.md
apps/
pet_factory/
pets/
source-assets/
work/
docs/
schemas/
tests/
reports/
releases/
archive/
```

Do not start a new Codex session inside:

- an old Codex image directory;
- `apps/desktop-pet/`;
- `source-assets/wangdulan/`;
- `work/wangdulan/`.

## Required Reading At Session Start

Read these files before acting:

1. `README.md`
2. `docs/PROJECT_STRUCTURE.md`
3. `docs/TOOL_RESPONSIBILITIES.md`
4. `docs/PET_PRODUCTION_PIPELINE.md`
5. `docs/CODEX_IMAGE_WORKFLOW.md`
6. `docs/WANGDULAN_VISUAL_LESSONS.md`
7. `source-assets/wangdulan/identity-reference/CHARACTER.md`
8. `work/wangdulan/walk16/CURRENT.md`
9. `work/wangdulan/walk16/tasks/I02/prompt.txt`
10. `work/wangdulan/walk16/tasks/I02/acceptance-checklist.md`

## First Task For New Codex

Generate only 2–3 `I02` candidate images for the `K2→K3` in-between task.

Write only to:

```text
source-assets/wangdulan/codex-output/walkRight/inbetweens/I02/
```

Do not perform QA, slicing, atlas assembly, app build, install, or release work.
